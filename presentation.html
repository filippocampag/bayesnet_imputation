Heart Disease Dataset Analysis

Giuseppe Pianeti, Filippo Campagnolo

The Heart Disease dataset

The Heart Disease dataset contains \(14\) attributes of \(920\) patients from \(4\) different clinics in the world: Cleveland, Hungarian, Switzerland and Long Beach. So far, machine learning applications on this dataset have concerned on the Cleveland clinc only, which consists of roughly \(1/3\) of the total samples, by trying to predict the presence of disease dis. In this work we attempt to include the other clinics and to deploy a sensitive analysis to compare the goodness of fit across different configurations of the dataset, using Random Forest (RF), Bayesian Additive Regression Trees (BART) and Bayesian Networks (BN) models.

  • cp: chest pain type (typical angina; atypical angina; non-anginal pain; asymptomatic)
  • trestbps: resting blood pressure
  • chol : serum cholestoral
  • fbs: fasting blood sugar > 120 mg/dl
  • restecg: resting electrocardiographic results (normal; having ST-T wave abnormality; showing probable or definite left ventricular hypertrophy)
  • thalach: maximum heart rate achieved
  • sex
  • exang: exercise induced angina
  • oldpeak: ST depression induced by exercise relative to rest
  • slope: the slope of the peak exercise ST segment during ECG (upsloping, flat, downsloping)
  • ca: number of major vessels colored by flourosopy (0-3)
  • thal: results of thallium stress test (normal; fixed defect; reversable defect)
  • dis: diagnosis of heart disease
  • age

  • As we can see, Cleveland is the only clinic reporting complete cases. This pattern clearly suggets MNAR missing data, i.e. the probability of being missing varies for reasons that are unknown to us, something likely linked to clinic procedures or protocols.
  • We decided to use BN for data imputation to preserve statistical power. However, this procedure still needs a complete dataset to learn the joint variables distribution \(\Pr(\mathbf{X})\), where \(\mathbf{X}=\{X_1,X_2,\dots,X_p,Y\}\), to be used for data imputation, thus an omogeneity assumption is needed.
  • By proceeding, we assume that the only MNAR data are the ones relative to slope, ca and thal and the other NA values are MCAR or at least MAR. Then, we assume omogeneity of these features over the clinic location and, therefore, the transferability of the estimated Bayesian Network from Cleveland to the other clinics. This assumption holds easily for slope, which is a parameter of a common, non-invasive diagnostic test. ca and thal refer instead to much more invasive and expensive tests that may be administered in a selective or different manner from one clinic to another; therefore, the assumption of homogeneity may not hold true.

Workflow

Data Imputation

for: every hyperparameter value:
  for: every fold:
    - train/test split
    - impute artificial NAs on test set
    - learning the structure "G" on train set
    - impute missing data with "G"
    - compute recostruction error
  average the errors => hyperparam score
choose hyperparam with mininmum error

learn the final DAG "FG" on the complete train data

learn the parameters of "FG"

impute missing data using "FG"

resulting DAG (for main analysis):

  • With the cross validation we look for the arc penalization parameter (k) and the maximum number of parents (maxp) that minimize the reconstruction error of artificially imputed missing data.
  • Then, we learn the final DAG and relative parameters on the complete train data.
  • Finally we can impute the real missing data on both the original train and test sets. For this purpose the bayes-lw method was desirable, but the large amount of missing data made the local distribution computations non feasible. Therefore, parents method or a custom function, which imputes missing data only relying on parent nodes and their order, were eventually used.

Models fitting and sensitivity analysis

  • We fit the following models: Random Forests, BART, a structure learned Bayesian Network, Naive Bayes and Augmented Naive Bayes.
  • Every model was firstly fitted on the complete imputed dataset with every predictor.
  • In order to assess the goodness of fit and the assumptions made, we deploy a sensitivity analyisis by training the same models with same procedures also on the complete cases dataset (1), a mean/mode imputed version of the data (2), a BN imputed reduced version of the full dataset without slope, ca and thal (3) and one excluding ca and thal only (4).

Performance measure: \[BA=\frac{sens +spec}{2}\] Balanced Accuracy, where \(sens\) is the sensitivity (or recall) while \(spec\) is the specificity.

Main Analysis (imputed data)

Fitted Models

  • For the RF model we used a \(10\) fold cross validation to assess the number of trees to be averaged and the maximum node size. The proportion of predictors to be used to grow the trees is set to \(\sqrt(p)\).
  • For the BART we validated the number of trees per iteration (ntree), the prior variance of \(f(x)\) (k) and two parameters for controlling the probability of split (base and power): \[\Pr(\text{split at depth }d) = \text{base} \cdot (1+d)^{-\text{power}}\]
  • With the BN model we learned the structure from the data, using the blacklist on the right and BIC penalization for the number of arcs. Then, we learned the parameters with bn.fit() and predicted the response.
  • For the fixed structure methods (NB and TAN) we simply learned the parameters and predicted the output class as above, using the predictors as hard evicende.

Sensitivity Analysis (1)

Four different configurations of the dataset we decided to analyze to evaluate the goodness of missing data imputation and the assumptions made.
These two scenarios were choosen to test, if the homogeneity assumption holds, whether the BN imputation led to an improvement of the performances:

Complete cases only (1)

  • In this scenario we followed the same exact workflow of the main analysis but filtering the original dataset with na.omit() and, thus, using almost only the Cleveland observations.
  • Using only complete instances, data imputation is not needed and the relative DAG was not computed
  • If the homogeneity assumption holds and, thus, the dag learned on cleveland can be transferred on other clinics, this scenario tells us whether including imputed incomplete cases brings any improvement over the performances.

Mean/mode imputation (2)

  • Here we test a different and more naive way for imputing missing data based on mean for continuous variables and mode for discrete variables. Being a much more rude and straighforward method we expect worst performance on the test set.
  • Using only complete instances, data imputation is not needed and the relative DAG was not computed
  • Always if homogeneity assumption holds, we expect to see worst accuracy performances, being MM imputation a much more rude and straighforward method of imputing data.
